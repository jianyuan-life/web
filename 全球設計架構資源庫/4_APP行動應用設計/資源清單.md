## 📐 APP設計原則

| 原則 | 說明 |
|------|------|
| 拇指區域設計（Thumb Zone） | 考慮單手操作時拇指可觸及的區域 |
| 最小點擊目標（44x44pt） | iOS建議的最小可點擊區域大小 |
| 手勢設計（Gesture Design） | 滑動、捏合、旋轉等手勢的設計規範 |
| 離線優先設計（Offline First） | 考慮網路不穩定時的使用者體驗 |
| 漸進式揭露（Progressive Disclosure） | 分步驟展示資訊，避免使用者超載 |
| 一致性（Consistency） | 與平台設計語言保持一致 |
